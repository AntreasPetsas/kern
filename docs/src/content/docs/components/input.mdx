## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `id` | `string` | — | **Required.** Links input to its label. |
| `label` | `string` | — | **Required.** Visible label text. |
| `value` | `string` | `''` | Controlled value |
| `onChange` | `ChangeEvent handler` | — | Change handler for controlled use |
| `placeholder` | `string` | — | Hint text — supplementary, not a label replacement |
| `helperText` | `string` | — | Hint below the field; hidden when `errorMessage` is set |
| `errorMessage` | `string` | — | Validation error; switches to error state |
| `disabled` | `boolean` | `false` | Prevents interaction |
| `required` | `boolean` | `false` | Adds `aria-required` and visual `*` indicator |
| `type` | `string` | `'text'` | HTML input type |
| `autoComplete` | `string` | — | HTML autocomplete attribute |

## Usage rules

### Do
- Always provide a **visible label** — use the `label` prop, not `placeholder` alone.
- Use `helperText` to set user expectations **before** they interact.
- Use `errorMessage` **after** validation — not before the user has had a chance to fill in the field.
- Use `type="email"`, `type="password"` etc. for correct mobile keyboard and browser auto-fill.
- Mark required fields with `required` and communicate the requirement in the label or `helperText`.

### Don't
- Don't use `placeholder` as a label — it disappears on input and fails contrast requirements.
- Don't show an error before the field has been interacted with.
- Don't rely on color alone to communicate error — the error message text is essential.
- Don't use `disabled` when `readonly` is more appropriate.
- Don't use for multi-line text — use a `<textarea>` instead.

## Accessibility

- **Role:** `textbox` (implicit — native `<input>` element)
- `<label htmlFor={id}>` always rendered and visible
- `aria-required="true"` when `required` is set
- `aria-invalid="true"` when `errorMessage` is present
- `aria-describedby` links to `helperText` or `errorMessage` element
- Error message uses `role="alert"` for immediate screen-reader announcement
