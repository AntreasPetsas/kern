Design tokens are the single source of truth for all visual values in Kern. They are authored once in `tokens/*.json` (W3C DTCG format) and compiled into CSS custom properties, TypeScript types, and a Tailwind preset by the build step.

## How to use

import { Tabs, TabItem } from '@astrojs/starlight/components';

<Tabs>
  <TabItem label="CSS custom properties">
    ```css
    /* Link build/css/variables.css, then reference --kern-* vars */
    .my-component {
      background-color: var(--kern-color-action-primary);
      padding: var(--kern-spacing-4);
      border-radius: var(--kern-radius-md);
      box-shadow: var(--kern-shadow-sm);
    }
    ```
  </TabItem>
  <TabItem label="TypeScript tokens">
    ```ts
    import { tokens } from '@kern-system/design-system/tokens';

    const primary = tokens.color.action.primary;   // "#2563eb"
    const spacing = tokens.spacing[4];             // "1rem"
    ```
  </TabItem>
  <TabItem label="Tailwind preset">
    ```js
    // tailwind.config.js
    import kernPreset from '@kern-system/design-system/tailwind';
    export default { presets: [kernPreset] };
    ```
    ```jsx
    <div className="bg-action-primary text-white p-4 rounded-md shadow-sm">…</div>
    ```
  </TabItem>
</Tabs>

## Color — semantic (prefer these)

## Radius

| Token | Value | Use |
|-------|-------|-----|
| `radius.none` | 0 | No rounding |
| `radius.sm` | 0.25rem | Inputs, small badges |
| `radius.md` | 0.5rem | Buttons, cards |
| `radius.lg` | 0.75rem | Modals, large cards |
| `radius.xl` | 1rem | Extra prominent |
| `radius.full` | 9999px | Pills, avatars |

## Shadow

| Token | Use |
|-------|-----|
| `shadow.none` | No shadow (flat style) |
| `shadow.sm` | Inputs, small cards |
| `shadow.md` | Cards, dropdowns |
| `shadow.lg` | Modals, popovers |
| `shadow.xl` | Overlays |
| `shadow.inner` | Pressed states, inset inputs |
